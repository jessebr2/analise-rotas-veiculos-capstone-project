Project 5: Análise de padrões de rotas de veículosÂ¶

IntroduçãoÂ¶

Neste projeto ser analisado o conjunto de dados de passagens de veculos por pontos de uma cidade. A base de dados contm a placa do veculo e a latitude e longitude do ponto de monitoramento. Esses dados se referem aos dias 14 e 15 de setembro de 2017. O objetivo deste projeto identificar padres de rotas de veculos, considerando para isso, a quantidade de vezes que cada veculo passou pelo ponto de controle em durante dois dias.

Poltica de privacidade e confidencialidade das informaes

Para que no seja infligida a privacidade dos donos dos veculos e a confidencialidade dos dados, tanto as placas quanto os dados dos pontos de controle foram cifrados para que se tornassem ininteligveis.

def create_df_placaporpontos(data_csv, local_column):
    lista_veiculos = set(data_csv.index)
    lista_locais = set(data_csv[local_column])
    data = pd.DataFrame(index = lista_veiculos, columns = lista_locais)
    data.fillna(0, inplace=True)
    for local in lista_locais:
        try:
            data_temp = pd.DataFrame(data_csv[data_csv[local_column] == local])
            data[local]=data[local].add(data_temp['count'], fill_value = 0)
        except:
            display(data_temp)
            data_temp.to_csv('data_temp.csv')
    return data

Implementação: PCAÂ¶

Alm de descobrir essas dimenses, a PCA tambm ir reportar a razo da varincia explicada de cada dimenso quanta varincia dentro dos dados explicada pela dimenso sozinha. Note que o componente (dimenso) da PCA pode ser considerado como um novo "feature" do espao, entretanto, ele uma composio do atributo original presente nos dados.

Redução da DimensionalidadeÂ¶

Ao utilizar um componente principal de anlise, um dos objetivos principais reduzir a dimensionalidade dos dados na realidade, reduzindo a complexidade do problema. Reduo de dimensionalidade tem um custo: Poucas dimenses utilizadas implicam em menor varincia total dos dados que esto sendo explicados. Por causo disso, a taxa de varincia explicada cumulativa extremamente importante para saber como vrias dimenses so necessrias para o problema. Alm disso, se uma quantidade significativa de varincia explicada por apenas duas ou trs dimenses, os dados reduzidos podem ser visualizados depois.

Pelo Array de variana explicada pode-se ver que com 30 dimenses 80% da variana dos dados explicada, portanto ser utilizada a reduo de dimensionalidade pelo PCA de 135 para 30 dimenses

def make_score(freq_target, freq_labels):
    a = freq_labels.shape[0]-freq_target.shape[0]
    freq_target = np.append(freq_target, np.zeros(a))
  
    target_ratio=np.divide(freq_target.astype(float), freq_labels)
    score = np.multiply(target_ratio, freq_target)
    max_score=np.max(score)
    
    return max_score

def test_n_clusters(data, data_target, n_clusters):
    
    max_ratio = 0

    clusterer = make_clusterer(data, n_clusters) #clusterizao
    labels = clusterer.labels_
    freq_labels = np.bincount(labels)
        
    data_target_pred = clusterer.predict(data_target) #predio dos dados alvo nos clusteres treinados
    freq_target = np.bincount(data_target_pred) #distribuio de frequncia por clusters dos dados alvo
      
    score = make_score(freq_target, freq_labels)
        
    return (score, clusterer)
